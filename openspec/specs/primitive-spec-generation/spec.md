# Specification

## Purpose

Behavioral requirements for the `PrimitiveSpecGenerator` tool (Agent 3, DS Bootstrap Crew, Task T3) and its integration within the agent pipeline. This spec covers: full primitive set coverage for all 11 React Native primitives (Box, Stack, HStack, VStack, Grid, Text, Icon, Pressable, Divider, Spacer, ThemeProvider), required YAML schema per primitive spec, W3C DTCG alias format for token bindings, composition rule encoding for the Component Factory Crew, and the CrewAI `BaseTool` contract binding Task T3 output to downstream Design-to-Code and Component Factory Crews (Phase 3).

Agent 3 (Primitive Scaffolding Agent) is a Tier 3 (Haiku) tool-invocation agent â€” its role is to call `PrimitiveSpecGenerator` deterministically with no creative reasoning, producing the canonical `specs/*.spec.yaml` set that all Phase 3 agents consume.

## Requirements

### Requirement: Full primitive set coverage

The `PrimitiveSpecGenerator` tool SHALL generate canonical spec YAMLs for all 9 composition primitives defined in PRD Â§3.3: Box, Stack, HStack, VStack, Grid, Text, Icon, Pressable, Divider, Spacer, and ThemeProvider. This covers the DS Bootstrap Crew (Agent 3) output contract for Task T3.

#### Acceptance Criteria

- [ ] `generate_all_primitive_specs(output_dir)` produces exactly 11 YAML files under `<output_dir>/specs/`: `Box.spec.yaml`, `Stack.spec.yaml`, `HStack.spec.yaml`, `VStack.spec.yaml`, `Grid.spec.yaml`, `Text.spec.yaml`, `Icon.spec.yaml`, `Pressable.spec.yaml`, `Divider.spec.yaml`, `Spacer.spec.yaml`, `ThemeProvider.spec.yaml`
- [ ] The function returns a `dict[str, str]` mapping each component name (e.g., `"Box"`) to the absolute path of its written spec file
- [ ] Calling the function twice with the same `output_dir` is idempotent â€” existing files are overwritten, no error is raised
- [ ] The `specs/` subdirectory is created if it does not exist

#### Scenario: All 11 spec files are written

- **GIVEN** an empty temporary directory `output_dir`
- **WHEN** `generate_all_primitive_specs(output_dir=str(output_dir))` is called
- **THEN** 11 files exist under `output_dir/specs/`
- **AND** the returned dict has exactly 11 keys matching the component names above
- **AND** each dict value is a string path to the corresponding file

#### Scenario: Idempotent re-run

- **GIVEN** `generate_all_primitive_specs` has already run and written all 11 files
- **WHEN** `generate_all_primitive_specs` is called again with the same `output_dir`
- **THEN** all 11 files exist with unchanged content
- **AND** no exception is raised

### Requirement: Required YAML fields per primitive spec

Every primitive spec YAML generated by `PrimitiveSpecGenerator` SHALL contain the following top-level fields: `component` (string), `description` (string), `props` (object), `tokenBindings` (array), `compositionRules` (object), `a11yRequirements` (object). This structure is used by the Design-to-Code Crew (Phase 3, Agents 13â€“16) and the Component Factory Crew (Agents 17â€“20) as their canonical input.

#### Acceptance Criteria

- [ ] Each generated YAML contains a `component` field with the exact PascalCase component name (e.g., `"Box"`, `"HStack"`)
- [ ] Each generated YAML contains a `description` string summarizing the primitive's role
- [ ] Each generated YAML contains a `props` object where each key is a prop name and each value has at minimum a `type` and `required` field
- [ ] Each generated YAML contains a `tokenBindings` array (may be empty `[]` for layout-only primitives)
- [ ] Each generated YAML contains a `compositionRules` object with at minimum `allowedChildren` (array or `"*"`) and `forbiddenNesting` (array, may be empty)
- [ ] Each generated YAML contains an `a11yRequirements` object with at minimum `role` (string or `null`), `focusable` (boolean), and `ariaAttributes` (array, may be empty)

#### Scenario: Box spec has all required fields

- **GIVEN** `generate_all_primitive_specs` has been called
- **WHEN** `Box.spec.yaml` is loaded with `yaml.safe_load`
- **THEN** the document contains keys: `component`, `description`, `props`, `tokenBindings`, `compositionRules`, `a11yRequirements`
- **AND** `component` equals `"Box"`

#### Scenario: Pressable spec has focusable=true

- **GIVEN** `generate_all_primitive_specs` has been called
- **WHEN** `Pressable.spec.yaml` is loaded
- **THEN** `a11yRequirements.focusable` is `true`
- **AND** `a11yRequirements.role` is a non-null string (e.g., `"button"` or generic interactive role)

#### Scenario: Divider and Spacer have empty tokenBindings

- **GIVEN** `generate_all_primitive_specs` has been called
- **WHEN** `Divider.spec.yaml` and `Spacer.spec.yaml` are loaded
- **THEN** both have `tokenBindings` as an empty list `[]`

### Requirement: Token binding key convention

All token binding values in primitive spec YAMLs SHALL use the W3C DTCG alias format: `{"$value": "{<semantic_token_key>}"}`. Token keys SHALL reference only semantic-tier tokens (keys beginning with `semantic.`) or global-tier dimension tokens (keys beginning with `scale.`). No hardcoded values (hex colors, px values, numeric literals) SHALL appear as token binding values. This requirement applies to the DS Bootstrap Crew (Agent 3) output and is enforced at validation time by the Token Validation Agent (Agent 7, Token Engine Crew, Phase 2).

#### Acceptance Criteria

- [ ] Every value in any `tokenBindings` array entry uses the DTCG alias format: `{"$value": "{ ... }"}`
- [ ] Token keys in binding values use dot-notation (e.g., `semantic.color.surface.default`)
- [ ] No `tokenBindings` entry contains a bare hex color, px value, or numeric literal as its `$value`
- [ ] `ThemeProvider.spec.yaml` has `tokenBindings: []` (ThemeProvider consumes no design tokens â€” exception documented per p05 spec)

#### Scenario: Text spec has semantic color token binding

- **GIVEN** `Text.spec.yaml` is generated and loaded
- **WHEN** the `tokenBindings` array is inspected
- **THEN** at least one entry references a semantic color token for text color (e.g., `semantic.color.text.primary`)
- **AND** the `$value` string starts with `{` and ends with `}`

#### Scenario: No hardcoded values in any spec

- **GIVEN** all 11 spec YAMLs are generated
- **WHEN** every `tokenBindings` array across all specs is inspected
- **THEN** no `$value` field contains a raw color string matching the pattern `#[0-9a-fA-F]{3,8}`
- **AND** no `$value` field contains a CSS length literal (e.g., `"16px"`, `"1rem"`)

### Requirement: Composition rules encoding

Primitive spec YAMLs SHALL encode composition rules that allow the Component Factory Crew (Agent 19, Composition Validation Agent) to verify that components compose from the primitive set correctly. Layout containers (Box, Stack, HStack, VStack, Grid) SHALL declare `allowedChildren: "*"`. Leaf primitives (Text, Icon, Divider, Spacer) SHALL declare `allowedChildren: []`. Interactive primitives (Pressable) SHALL declare their allowed content model explicitly.

#### Acceptance Criteria

- [ ] `Box.spec.yaml`, `Stack.spec.yaml`, `HStack.spec.yaml`, `VStack.spec.yaml`, `Grid.spec.yaml` each have `compositionRules.allowedChildren: "*"`
- [ ] `Text.spec.yaml`, `Icon.spec.yaml`, `Divider.spec.yaml`, `Spacer.spec.yaml` each have `compositionRules.allowedChildren: []`
- [ ] `Pressable.spec.yaml` has `compositionRules.allowedChildren` set to a non-empty value (children allowed, but not `Pressable`)
- [ ] `Pressable.spec.yaml` has `compositionRules.forbiddenNesting` containing `"Pressable"` (no nested Pressable elements per Â§3.3 composition rules)
- [ ] `ThemeProvider.spec.yaml` has `compositionRules.allowedChildren: "*"` (wraps the entire app tree)

#### Scenario: Box allows any children

- **GIVEN** `Box.spec.yaml` is loaded
- **WHEN** `compositionRules.allowedChildren` is read
- **THEN** the value is the string `"*"`

#### Scenario: Pressable forbids nested Pressable

- **GIVEN** `Pressable.spec.yaml` is loaded
- **WHEN** `compositionRules.forbiddenNesting` is read
- **THEN** the list contains `"Pressable"`

#### Scenario: Text is a leaf node

- **GIVEN** `Text.spec.yaml` is loaded
- **WHEN** `compositionRules.allowedChildren` is read
- **THEN** the value is an empty list `[]`

### Requirement: PrimitiveSpecGenerator CrewAI Tool contract

The `PrimitiveSpecGenerator` class SHALL be a `BaseTool` subclass importable from `daf.tools.primitive_spec_generator`. Its `_run()` method SHALL accept an `output_dir` string argument, call `generate_all_primitive_specs(output_dir)`, and return a JSON-serializable summary string reporting how many specs were written and to which paths. This tool is wired into Agent 3 (DS Bootstrap Crew, Task T3) in the DS Bootstrap Agent module.

#### Acceptance Criteria

- [ ] `from daf.tools.primitive_spec_generator import PrimitiveSpecGenerator` succeeds
- [ ] `PrimitiveSpecGenerator` is a subclass of `crewai.tools.BaseTool`
- [ ] `PrimitiveSpecGenerator().run(output_dir="<path>")` calls `generate_all_primitive_specs` and returns a string summary
- [ ] The tool's `name` field is `"PrimitiveSpecGenerator"`
- [ ] The tool's `description` field accurately describes its function for LLM agent consumption

#### Scenario: Tool run produces summary string

- **GIVEN** a `PrimitiveSpecGenerator` instance
- **WHEN** `tool.run(output_dir=str(tmp_path))` is called
- **THEN** it returns a non-empty string (e.g., `"Generated 11 primitive specs in <path>/specs/"`)
- **AND** 11 spec YAML files now exist under `tmp_path/specs/`

#### Scenario: Tool is importable and correctly typed

- **GIVEN** `from daf.tools.primitive_spec_generator import PrimitiveSpecGenerator`
- **WHEN** `PrimitiveSpecGenerator` is inspected
- **THEN** it is a subclass of `BaseTool`
- **AND** `PrimitiveSpecGenerator.name` equals `"PrimitiveSpecGenerator"`
